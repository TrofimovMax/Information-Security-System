# BPMN-описание: Модуль «Заявки на обучение»

## 1. Пулы и Дорожки

### Пул «Портал»
Основной пул, охватывающий внутренние роли системы. Разбит на дорожки:

| Дорожка | Описание |
|---|---|
| **Заявитель** | Сотрудник, ответственный за подачу заявки на обучение (до 500 чел.) |
| **Согласующий** | Сотрудник, уполномоченный проверять и утверждать заявки |
| **Система (Портал)** | Автоматические действия платформы: уведомления, смена статусов, передача данных |

### Пул «СДО»
Внешняя Система Дистанционного Обучения. Принимает согласованные заявки по API и предоставляет список актуальных курсов.

### Пул «Почтовая служба»
Внешний сервис email-рассылок. Получает запросы от Портала и доставляет уведомления заинтересованным лицам.

### Пул «SSO / Система аутентификации»
Внешняя система единого входа. Проводит аутентификацию и выдаёт токен авторизации.

## 2. События

### Начальные события (Start Events)

| ID | Тип | Дорожка | Описание |
|---|---|---|---|
| SE-01 | Начальное (None) | Заявитель | Заявитель решает подать заявку на обучение — инициирует процесс |
| SE-02 | Начальное сообщение (Message) | Согласующий | Получение уведомления о новой заявке, ожидающей согласования |

### Промежуточные события (Intermediate Events)

| ID | Тип | Дорожка | Описание |
|---|---|---|---|
| IE-01 | Промежуточное сообщение — получение (Message Catch) | Заявитель | Получение уведомления об изменении статуса заявки (отклонена / на корректировку) |
| IE-02 | Промежуточное сообщение — получение (Message Catch) | Заявитель | Получение уведомления о финальном статусе (одобрена / отклонена окончательно) |
| IE-03 | Промежуточное сообщение — отправка (Message Throw) | Система (Портал) | Отправка запроса в Почтовую службу при любом изменении статуса заявки |

### Конечные события (End Events)

| ID | Тип | Дорожка | Описание |
|---|---|---|---|
| EE-01 | Конечное (None) | Заявитель | Заявка окончательно отклонена — процесс для заявителя завершён |
| EE-02 | Конечное сообщение (Message) | Система (Портал) | Заявка передана в СДО — процесс успешно завершён |
| EE-03 | Конечное (None) | Почтовая служба | Email-уведомление отправлено адресатам |

## 3. Процессы: Задачи и Подпроцессы

### Дорожка «Заявитель»

| ID | Тип | Название | Описание |
|---|---|---|---|
| T-01 | Задача (User Task) | Получить шаблон Excel-файла | Заявитель скачивает актуальный шаблон для формирования списка участников |
| T-02 | Задача (User Task) | Получить список актуальных курсов | Заявитель просматривает курсы, доступные в СДО |
| SP-01 | Подпроцесс | Создать заявку | Включает альтернативные способы создания заявки (см. ниже) |
| T-03 | Задача (User Task) | Загрузить Excel-файл | Заявитель загружает заполненный файл со списком участников |
| T-04 | Задача (User Task) | Заполнить веб-форму | Заявитель вручную вводит список участников через интерфейс Портала |
| T-05 | Задача (User Task) | Дополнить веб-форму вручную | После загрузки Excel заявитель при необходимости корректирует данные в форме |
| T-06 | Задача (User Task) | Скорректировать заявку | Заявитель вносит изменения по комментарию согласующего и повторно отправляет заявку |

**Декомпозиция подпроцесса SP-01 «Создать заявку»:**

Внутри подпроцесса располагается шлюз GW-01, от которого расходятся два пути:
- Путь A: T-03 → T-05 (загрузка файла с опциональным дополнением)
- Путь B: T-04 (заполнение веб-формы)

Оба пути сходятся в шлюзе GW-02 перед передачей заявки на согласование.

### Дорожка «Согласующий»

| ID | Тип | Название | Описание |
|---|---|---|---|
| T-07 | Задача (User Task) | Просмотреть список заявок | Согласующий открывает очередь заявок, ожидающих проверки |
| T-08 | Задача (User Task) | Просмотреть заявку | Детальное изучение состава участников и курсов |
| T-09 | Задача (User Task) | Одобрить заявку | Согласующий подтверждает заявку; комментарий не требуется |
| T-10 | Задача (User Task) | Отклонить заявку | Согласующий отклоняет заявку; комментарий опционален |
| T-11 | Задача (User Task) | Отправить на корректировку | Согласующий возвращает заявку с обязательным комментарием |
| T-12 | Задача (User Task) | Направить заявку в СДО | Согласующий инициирует передачу одобренной заявки в СДО по API |

### Дорожка «Система (Портал)»

| ID | Тип | Название | Описание |
|---|---|---|---|
| T-13 | Задача (Service Task) | Изменить статус заявки | Автоматическое обновление статуса в БД Портала |
| T-14 | Задача (Service Task) | Сформировать портальное уведомление | Создание уведомления внутри Портала для заявителя и/или согласующего |
| T-15 | Задача (Service Task) | Отправить заявку в СДО (API) | Автоматическая передача данных заявки в СДО через API |

### Дорожка «СДО»

| ID | Тип | Название | Описание |
|---|---|---|---|
| T-16 | Задача (Service Task) | Предоставить список актуальных курсов | СДО возвращает Порталу актуальный каталог курсов |
| T-17 | Задача (Service Task) | Принять заявку на обучение | СДО фиксирует заявку и добавляет участников в соответствующие курсы |

### Дорожка «Почтовая служба»

| ID | Тип | Название | Описание |
|---|---|---|---|
| T-18 | Задача (Service Task) | Отправить email-уведомление | Рассылка письма заинтересованным лицам об изменении статуса заявки |

## 4. Шлюзы

| ID | Тип | Расположение | Вопрос / Условие |
|---|---|---|---|
| GW-01 | Эксклюзивный (XOR) | SP-01 / Заявитель | Каким способом создаётся заявка? |
| GW-02 | Эксклюзивный (XOR) — слияние | SP-01 / Заявитель | Объединение путей после создания заявки перед отправкой на согласование |
| GW-03 | Эксклюзивный (XOR) | Согласующий | Каково решение по заявке? |
| GW-04 | Эксклюзивный (XOR) — слияние | Согласующий | Объединение путей «одобрено» / «на корректировку (повторно)» перед направлением в СДО |
| GW-05 | Параллельный (AND) | Система (Портал) | Одновременная отправка: портальное уведомление + запрос в Почтовую службу |

## 5. Потоки управления (Sequence Flow)

Основная последовательность:

```
SE-01 → T-01 → T-02 → SP-01(GW-01) → [Path A | Path B] → GW-02 → T-13 → GW-05 → T-14 / IE-03
                                                                             ↓
                                                                    SE-02 → T-07 → T-08 → GW-03
```

Внутри SP-01:
- `GW-01 --[Excel]--> T-03 --> T-05 --> GW-02`
- `GW-01 --[Форма]--> T-04 --> GW-02`

После GW-03:
- `GW-03 --[Одобрить]--> T-09 --> T-12 --> T-15 --> EE-02`
- `GW-03 --[Отклонить]--> T-10 --> T-13 --> GW-05 --> EE-01`
- `GW-03 --[На корректировку]--> T-11 --> T-13 --> GW-05 --> IE-01 (Заявитель) --> T-06 --> GW-02 (повтор)`

## 6. Условные потоки управления (Conditional Sequence Flow)

| ID | От | К | Условие |
|---|---|---|---|
| CSF-01 | GW-01 | T-03 | Заявитель выбирает загрузку Excel-файла |
| CSF-02 | GW-01 | T-04 | Заявитель выбирает ручное заполнение веб-формы |
| CSF-03 | T-03 | T-05 | После загрузки файла заявитель хочет дополнить форму вручную (опциональный путь) |
| CSF-04 | T-03 | GW-02 | После загрузки файла заявитель не дополняет форму |
| CSF-05 | GW-03 | T-09 | Согласующий принимает решение «Одобрить» |
| CSF-06 | GW-03 | T-10 | Согласующий принимает решение «Отклонить» |
| CSF-07 | GW-03 | T-11 | Согласующий принимает решение «На корректировку» |
